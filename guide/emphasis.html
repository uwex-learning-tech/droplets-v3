Emphasis

Feeling bold? Be strong!

Bold

For representing important content with a heavier font weight, use the <strong> element. Changing the importance of a piece of text with <strong> element does not and should not change the meaning of the sentence.

Notes: per HTML5 specification, the <b> element should be used as a last resort.

See the Pen Bold by Ethan Lin (@Lin87) on CodePen.

Italics

For emphasizing content with a slightly slanted font style, use the <em> element. The placement of <em> element changes the meaning of the sentence.

Notes: per HTML5 specification, the <i> element should be used as a last resort.

See the Pen Italics by Ethan Lin (@Lin87) on CodePen.

Notes: the differences between <b> vs <strong> and <i> vs <em> are typographic effects (presentational) and semantics. The HTML5 specification redefines the use of <b> and <i> to have some semantic functions and, optionally, typographic effects that are specified and dependent on a CSS style definition. For more information on the use of <b> and <i> elements, please visit W3C Internationalization (i18N) Activity.

Underline

For emphasizing content with an underline, use a <span> element with an underline class.

See the Pen Underline by Ethan Lin (@Lin87) on CodePen.

Tip: do not over use underlines. Underlines are frown upon because they are the web browser default for links. Making a regular text looks like a link give users a bad case of clicky-clicky frustration.

Notes: do not use <u> or <ins> element for underlining. The <u> element has been redefined in HTML5 for other purpose with semantic meanings. The <ins> element represents that a text has been inserted into a document. By default, web browser renders both <u> and <ins> elements with an underline; however, the style of each element is solely dependent on the CSS style definition.

Abbreviation

JavaScript component script file is required for abbreviation to work.

Implementation of <abbr> element for abbreviation and acronyms to show the expanded version on hover. The <abbr> element must have a title attribute, which holds the full meaning of the abbreviation or acronyms.

See the Pen JDovI by Ethan Lin (@Lin87) on CodePen.

Small Text

For de-emphasizing or a smaller font size, use the <small> element. The <small> element has an inline position and can be nested.

See the Pen Small Text by Ethan Lin (@Lin87) on CodePen.

Colored Text

Convey meanings through colors by using the following classes: text-info (light-blue), text-success (green), text-warning (yellow), and text-danger (red).

Please refer to the best practice guide for the use of colored text.

See the Pen Colored Text by Ethan Lin (@Lin87) on CodePen.

Code Text

Differentiate code text from regular text with the use of <code> element. The <code> element has an inline position and can be nested.

See the Pen Code Text by Ethan Lin (@Lin87) on CodePen.
